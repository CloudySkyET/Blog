题目描述

玩一种类似汉诺塔的游戏，只不过围成了一圈，你只能顺时针走。分别求将 nnn 个盘子移动到 222 柱和 333 柱的最小步数，多组询问。

如：两个盘子移到 222 柱：111 盘移两步到 333 柱，222 盘移一步到 222 柱，111 盘移两步 2→3→12 \to 3 \to 12→3→1。最终答案为 555。

思路

一道找规律题。

考虑如果想要移到 222 柱，一定是先把 n−1n - 1n−1 移两步，把第 nnn 个移一步，然后再把 n−1n - 1n−1 移两步。

而如果要移到 333 柱，则需要先把 n−1n - 1n−1 移两步，把第 nnn 移一步，再把 n−1n - 1n−1 移一步，再把第 nnn 移一步。再把 n−1n - 1n−1 移两步。

因为只要是移两步或者移一步，从哪个柱开始是不重要的。所以可以分别设将 nnn 个盘子移 111 步和移 222 步的最小步数为 an,bna_n, b_nan​,bn​。

转移式就是 an=2×bn−1+1, bn=2×bn−1+an−1+2a_n = 2 \times b_{n - 1} + 1,\ b_n = 2 \times b_{n - 1} + a_{n - 1} + 2an​=2×bn−1​+1, bn​=2×bn−1​+an−1​+2

发现这个东西可以套矩阵快速幂。但是复杂度过不去，俩字分块\Large{\text{分块}}分块。专业术语光速乘。

时间复杂度 O(n×sz3)O(\sqrt n \times sz^3)O(n​×sz3) 预处理 O(1)O(1)O(1) 查询。

T1 我永远喜欢月

题目描述

给出两个 n×mn\times mn×m 的 010101 矩阵 A,BA,BA,B。一次操作可以指定 AAA 中一个为 000 的位置，将它变为 111 ，同时将该行该列其它元素变为 000 。求是否可能通过这些操作将 AAA 变成 BBB。

思路

大结论题。

先判完全相等。

你发现如果 BBB 矩阵中某行某列有超过 111 个 111，那么这行/列锁了。同时 111 所对的列/行锁了。

如果锁定点与 AAA 不同，那么判定失败。（考场上判错了）

把锁定行/列从图里扣掉，如果剩下的位置 AAA 没有 000 或者 BBB 没有 111 判定失败，否则成功。

T3 你猜是不是找规律

找个*规律

题目描述

优秀的出题人的题面从来都是一句话：求有多少长为 nnn 的排列交换任意两个数字不超过 kkk 次即能使之有序。

思路

虽然有序有争议吧，不过看样例还是能看出来的。

首先考虑 kkk 次变有序不太好做，转成对一个有序序列做 kkk 次能变成什么序列，但是你要保证 kkk 次没有反复操作。

考场上突然灵光乍现想到了可以 dpdpdp 。然后又瞎写了个转移方程，看了看没准有戏，模了组数据发现应该是对的。

设 fi,jf_{i, j}fi,j​ 表示当前枚举到 iii 号点，交换了 jjj 次。那么转移是不是就是：

fi,j=fi−1,j−1×(i−1)+fi−1,jf_{i, j} = f_{i - 1, j - 1} \times (i - 1) + f_{i - 1, j} fi,j​=fi−1,j−1​×(i−1)+fi−1,j​

意思就是如果当前位置要交换就从前面 i−1i - 1i−1 个位置随便挑一个来换。由于你前面的 j−1j - 1j−1 次操作只是在前 i−1i - 1i−1 个数里进行的，所以把第 iii 个数换进去一定不会造成反复。满足条件。

然后这个复杂度是过不去的，但是你发现这个转移可以用斯特林数来优化。

T1 工作

题目描述

有 nnn 个工人，第 iii 个工人的初始效率值为 aia_iai​ 。

有 mmm 个工作，第 iii 个工作需要恰好 kik_iki​ 个工人来完成。

思路

想一想应该就知道，每个任务应该找到的是前 kik_iki​ 的工人。
