网络流及 Dinic 学习笔记

网络流是 OI 中一种非常招人痛恨的算法，因其较难理解且理解后也做不对题。

网络流的基础知识听起来很抽象，但实际上并不是非常难理解。它分为 网络 和 流。（不然为啥子叫网络流？）

网络

网络就是一张满足一些性质的图。

  • 首先，它是一张有向图。
  • 图中的每条边都有一个 ccc 值，代表这条边的容量。
    可表示为 c(u,v)(u,v∈V)c(u,v)_{(u,v\in V)}c(u,v)(u,v∈V)​
  • 它具有两个特殊的点，源点和汇点。
    （这两个点一般都不是题目给出的。）

流

流是一个函数。流函数可表示为： f(u,v)(u,v∈V)f(u,v)_{(u,v\in V)}f(u,v)(u,v∈V)​ 具体的性质分为以下三条，（这些性质名字看起来很高深，但类比水流很容易理解）

  1. 容量限制：f(u,v)≤c(u,v)f(u,v)\le c(u,v)f(u,v)≤c(u,v)
  2. 斜对称性：f(u,v)=−f(v,u)f(u,v) = - f(v,u)f(u,v)=−f(v,u)
  3. 流守恒性：

∀x∈V,且x∉{s,t},∑(u,x)∈Ef(u,x)=∑(x,v)∈Ef(x,v)\forall x\in V,\text且 x \notin\{s,t\},\sum_{(u,x)\in E} f(u,x) = \sum_{(x,v)\in E} f(x,v) ∀x∈V,且x∈/{s,t},(u,x)∈E∑​f(u,x)=(x,v)∈E∑​f(x,v)

(通俗理解一下，就是每个点都不会有“积水”)

用网络流能解决的问题

网络流常见的问题可以概括理解为以下三种：

最大流问题：求一张网络从源点到汇点的最大流量。（这个问题也可以转变为二分图最大匹配问题。）

最小割问题：删掉总流量为 XXX 的边使整张图不连通，最小的 XXX 。仔细推一下就能发现，对于一张图，最小割和最大流其实得到的答案其实是一样的。

费用流问题：全称为最大费用最大流或者最小费用最大流。顾名思义，就是在网络中的每条边中加入一种属性，也就是这条边的费用。你在这条边上的花费就是 这条边的费用×这条边的流量\text{这条边的费用} \times \text{这条边的流量}这条边的费用×这条边的流量 。要求优先满足最大流量的同时满足最大费用。

解决网络流问题的算法

DinicDinicDinic 算法

时间复杂度：O(n2m)O(n^2m)O(n2m) （在二分图上是 O(nm)O(\sqrt nm)O(n​m) ）

虽然说这个复杂度看起来非常劣，但网络流算法的复杂度往往比这个上界低很多。我们只需要记住这个算法可以处理的数据范围是 104∼10510^4 \sim 10^5104∼105 即可。

具体实现：

  • 每次先 BFSBFSBFS 将这个残量网络（即每条边的容量减掉流量）构建为分层图。
  • 后用 DFSDFSDFS 在这个分层图上同时找出多条增广路。
  • 将增广路上的边流量减小最大可减小剩余容量，同时将对应的反向边的剩余容量增加（方便后面“反悔”），这就要求我们建边时成对建边（听起来挺简单的？）
  • DinicDinicDinic 算法还有一个常见的优化。当前弧优化。也就是当前增广时如果有一条边已经被增广过或者不能被增广，那就将这条边在当前轮从边集中删除。

网络流建图套路小总结

  • 如果给定某些物品的初态以及终态，求某种极限转移，可以将源点连向初态费用为 0 ，终态连向汇点费用为 0，转移连成出边费用为代价。

    经典题：负载平衡。

  • 某些取数类问题，可以变成从起点走过物品到终点，其中走过某个物品就代表选了它。将每个物品拆成两个点，中间连一条容量为取数限制权值为物品价值的边。最后最大流即为答案。

    经典题：方格取数加强版。

  • 网格图相邻问题可以黑白交替转染色化成二分图，之后跑网络流。

    经典题：方格取数问题。

  • 某些阶段性问题可以构建分层图跑网络流。

    经典题：家园/星际转移问题。
