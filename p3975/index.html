P3975 [TJOI2015]弦论 后缀数组与后缀自动机的拓展应用（磕了一天的题）

题目描述

给定一个字符串 sss ，求排名为 kkk 的子串，分两种情况：

  1. 本质相同，位置不同算 111 种。
  2. 本质相同，位置不同算 多 种。

题目正解

SA/SAMSA/SAMSA/SAM

题目思路 - SASASA

对于 SASASA 来讲， 111 类相对来将好解一些。对于一个后缀 xxx，比他小的所有子串的个数为：

∑i=1x−1n−sa[i]+1−height[i]\sum_{i = 1}^{x - 1} n - sa[i] + 1 - height[i] i=1∑x−1​n−sa[i]+1−height[i]

这个东西理解起来就是，你把所有的后缀按照 rkrkrk 排成一列，那么对于每个后缀，他能进行的贡献即为 s[sa[i]∼sa[i]+height[i]],s[sa[i]∼sa[i]+height[i]+1],⋯ ,s[sa[i]∼n]s[sa[i]\sim sa[i] + height[i]], s[sa[i]\sim sa[i] + height[i] + 1], \cdots, s[sa[i]\sim n]s[sa[i]∼sa[i]+height[i]],s[sa[i]∼sa[i]+height[i]+1],⋯,s[sa[i]∼n] 。原因就是对于它和上一段的 lcplcplcp ，一定已经在上一段或更早的时候被统计到了。所以可以直接线性扫一遍直到排名刚好为 kkk。

222 类相对来说困难一些。因为对于它和后面的后缀的 lcplcplcp，没有办法当前串统计到。

但是，正难可以反，我们可以发现，给定一个知道起始位置为 ppp 和长度 www 的串，求他的排名相对来讲简单一些。rkrkrk 在它前面的后缀，对答案的贡献为 ∑n−sa[i]+1\sum n - sa[i] + 1∑n−sa[i]+1 （因为本质不同算多个，所以不去要减 heightheightheight）。对于它后面的串，对答案的贡献为 ∑min⁡(w,lcp(p,i))\sum\min(w, lcp(p, i))∑min(w,lcp(p,i)) 后面的 lcplcplcp 只需要对 heightheightheight 预处理 STSTST 表就能做到 O(1)O(1)O(1) 查询。最终的式子为：

现在的关键就是如何求 ppp 和 www ，做法就是二分情况 111 下的排名。单调性显然。

所以整个过程就是 二分排名（情况 111） →\to→ 求出对应的子串 →\to→ 求出排名（情况 222），比对。 最终答案要的是 rk≥krk \ge krk≥k 的最小子串。

题目思路 - SAMSAMSAM

对于 SAMSAMSAM 来讲，只要思路对了两种情况差不多。

构建出 SAMSAMSAM，对于每个点 xxx ，统计出以它结尾的子串个数，记为 siz[x]siz[x]siz[x] ，利用 sizsizsiz 统计出经过它的所有子串个数，记为 sum[x]sum[x]sum[x] 。

至于如何去求 sizsizsiz 和 minminmin ，大概有两种方法，一种是常规的倒着连 failfailfail 然后跑 dfsdfsdfs 或 bfsbfsbfs ，而另一种则是对 lenlenlen 从小到大进行排序，倒着扫更新（当然不能直接对 lenlenlen 排序，而是要建立一个映射关系。） 这样就能保证更新 fail[x]fail[x]fail[x] 时已经更新过 xxx 了。对于条件 111 在求 sumsumsum 之前将每个点的 sizsizsiz 赋成 111 即可。

递归进行输出，每到一个点 a∼za\sim za∼z 枚举转移边，比较 kkk 和 sum[sam[p][i]]sum[sam[p][i]]sum[sam[p][i]] 的大小关系。

  • k>sum[sam[p][i]]k > sum[sam[p][i]]k>sum[sam[p][i]] 说明要找的点不经过这个转移边，k=k−sum[sam[p][i]]k = k - sum[sam[p][i]]k=k−sum[sam[p][i]]。
  • k≤sum[sam[p][i]]k \le sum[sam[p][i]]k≤sum[sam[p][i]] 说明要找的点需要经过这个转移边。输出 iii 并递归。结束后直接回溯防止重复输出。

总结

通过写这道题收获不小，大概掌握了后缀数据结构处理第 kkk 大子串的方法。
