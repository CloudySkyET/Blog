T1. Divisors

题目大意：

给定一个长度为 mmm 的数列 a1,a2,…ama_1, a_2, \dots a_ma1​,a2​,…am​，和区间 [1,n][1, n][1,n]。求 ∀k∈[0,m]\forall k \in [0, m]∀k∈[0,m], 在区间 [1,n][1, n][1,n] 里有多少正整数是 aaa 中恰好 kkk 个数的约数。

题目正解：

因数分解，STL + map

题目思路：

首先看到约数个数的问题，就想到了因数分解。

枚举每个数 aia_iai​ 的约束 d1,d2,…dxd_1, d_2, \dots d_xd1​,d2​,…dx​。

用 mapmapmap 记录每个约数出现的次数，即 mapd=∑k=1n[d∣ak]map_d = \sum_{k = 1}^n[d|a_k]mapd​=∑k=1n​[d∣ak​]

同时记录所有因数的出现的次数（非重叠） cnt。

最后用迭代器遍历一遍 map, 统计答案。ansx=∑[mapd==x]ans_x = \sum [map_d == x]ansx​=∑[mapd​==x]

当 k = 0 时，答案即为非因数个数 n - cnt。

同时注意对 axa_xax​ 进行因数拆分时注意判断 d>nd > nd>n。

时间复杂度：

O(mlogm×axax)O(mlogm\times a_x\sqrt a_x)O(mlogm×ax​a​x​)

T2.Market

题目大意：

共有 nnn 种物品，个数为 111，体积为 cic_ici​，价值为 viv_ivi​，开放时间为 tit_iti​，共有 mmm 次询问，每次询问在时间 TiT_iTi​ 时，体积为 ViV_iVi​ 的背包所能容纳的最大价值。

其中 cic_ici​ 远大于 viv_ivi​

题目正解：

01 背包 + 离线 + 交换主维

题目思路：

首先一眼就能看出来的是 01 背包，然后我们要考虑优化。

每次询问找到可用的所有物品并跑一边背包肯定是不现实的，所以就用到了离线排序思想。

将物品和询问按照时间排序。每次更新背包只要跑一边新加入的物品。

然后发现这样只能拿到 60’ 因为 c 太大，数组开不下。这时我们就要用到交换两维的思想，枚举 v，让 c 尽可能小。然后再二分 f 数组，求得 最后一个大于等于 ViV_iVi​ 的 c 值所对应的下标

时间复杂度：

O(nv)O(nv)O(nv)

T3.披风

题目大意：

给定一张有向图，每个点有点权。试找到一条路径，使得该路径上的点权最大值减去点权最小值最大，问这个差最大是多少。

题目正解：

tarjan 缩点 + 拓扑 DP \ 记忆化搜索

题目思路：

看到有向图关于链的问题就能大概想到拓扑排序或者 爆搜 记忆化搜索 。它的前置自然是缩点构造 DAG。

时间复杂度：

O(n+m)O(n + m)O(n+m)

T4.子弹

题目大意：

nnn 个点，每个点两个数共 2×n2 \times n2×n 个数，可以交换任意两个数，要求最小的修改数使每个点两个数的差值比 c 小。

题目正解：

搜索

题目思路：

首先可以确定的是不存在某个点中两个数都是从其他点换过来的。这样肯定不优。

然后我们就可以搜索从 1 开始让每个点依次满足条件。最终得到答案，可以用迭代加深优化。

时间复杂度：

O(玄学)O(\text{玄学})O(玄学)
