因为能做的题不多，所以用一篇简单的对常中模拟赛中能做懂的题进行一些总结。

D1T1 count

题目描述

有一张 nnn 个点 mmm 条边的无向图。

记 C1C_1C1​ 为这个图导出子图为 K4K_4K4​ 的数量, C2C_2C2​ 为补图导出子图为 K4K_4K4​ 的数量（K4K_4K4​ 是 444 个点的完全图）。

请你求出 C1−C2C_1 - C_2C1​−C2​ 的值。

思路

一个容斥。

考虑对 C1C_1C1​ 进行容斥。设 FiFiFi 表示对于 444 个点，只对 iii 条边进行限制的导出子图方案数。可以发现：

C1=∑i6(−1)i+1Fi=−F0+F1−F2+F3−F4+F5−F6C_1 = \sum_i^6 (-1)^{i + 1} Fi = - F0 + F1 - F2 + F3 - F4 + F5 - F6 C1​=i∑6​(−1)i+1Fi=−F0+F1−F2+F3−F4+F5−F6

然后可以发下 C2C_2C2​ 正好和 F6F6F6 有关系，可以消掉，接下来就好办了。

分别求出 F0∼F5F0\sim F5F0∼F5 即可。

对于 F0F0F0，随便选 444 个点，再除以他们的排列数。

n * (n - 1) * (n - 2) * (n - 3) / 24

对于 F1F1F1，其中两个点限制一条边，另外两个点随便选。

m * (n - 2) * (n - 3) / 2

对于 F2F2F2，分两类考虑，分别是两条边有交点和两条边无交点。

对于有交点，枚举每一个点，分别从它的度数中选取两个作为边，这两条边无序，所以需要 /2/2/2，剩下一个点随便选。

(n - 3) * d[i] * (d[i] - 1) / 2

对于无交点，两条边正好 444 个端点。

只要枚举每条边，分别扣掉和它相邻的边即可。同样，边无序，需要 /2/2/2。

(m - (d[e[i].u] + d[e[i].v] - 1)) / 2

对于 F3F3F3 ，一共有三类情况：

  1. 这 333 条边构成了三元环，这边是一个经典的 mmm\sqrt mmm​ 的三元环计数问题。最后再 ×(n−6)\times (n - 6)×(n−6) 代表环外的那个点。

  2. 这 333 条边构成了一个菊花。

    类似于 F2F2F2 的有交点，分别枚举每个点的 333 条边，最后 /6/6/6 。

    d[i] * (d[i] - 1) * (d[i] - 2) / 6

  3. 这 333 条边构成了一条链。

    直接枚举每条边，对它的两个端点分别再连一条边。

    (d[e[i].u] - 1) * (d[e[i].v] - 1)

对于 F4F4F4，有两类情况，分别是四元环和三元环链一个点。

对于 F5F5F5，只有一种情况，两个三元环有一个公共点。

D3T1 拆分

题目描述

现有一整数 nnn，求满足以下条件的整数序列个数：

  • a1+a2+⋯+am=na_1 + a_2 + \cdots + a_m = na1​+a2​+⋯+am​=n
  • 0<a1≤a2≤⋯≤am0 < a_1 ≤ a_2 ≤ \cdots ≤ a_m0<a1​≤a2​≤⋯≤am​
  • ∀i≠j,∣ai−aj∣≥k∀i \ne j, |ai − aj| ≥ k∀i=j,∣ai−aj∣≥k
  • hm mod p=1h_{m \bmod p} = 1hmmodp​=1

思路

不考虑 kkk 的限制的话 n2n^2n2 整数拆分即可。

fi,jf_{i, j}fi,j​ 表示划分了 iii 段，总和为 jjj 的方案数。

转移的话因为单调不降，所以有两种方法，把前面所有数都加 1 和另开一段 1。

fi,j=fi,j−i,fi−1,j−1f_{i, j} = f_{i, j - i}, f_{i - 1, j - 1} fi,j​=fi,j−i​,fi−1,j−1​

对于 kkk 的限制，直接垫高补齐就行了（如果要拆分成 iii 个数，那么只需要和为 n−i×(i−1)2×kn - \frac{i \times (i - 1)}{2} \times kn−2i×(i−1)​×k）。反而可以消掉一些复杂度，使它满足 O(nn)O(n\sqrt n)O(nn​)。

D3T2 集合

题目描述

现有 NNN 个集合 SiS_iSi​ ，每个集合可以表示成两个闭区间的并。

QQQ 次询问，每次询问 l,rl, rl,r，求最小的 xxx 满足 ∀l≤i≤r,x∈Si∀l ≤ i ≤ r, x ∈ S_i∀l≤i≤r,x∈Si​。

思路

一道非常妙的数据结构。

首先可以把询问离线下来，按照右端点排序，依次扫描，每到一个询问，就把 ri−1∼rir_{i - 1} \sim r_iri−1​∼ri​ 中所有集合拿出来更新。

对于每个集合 SiS_iSi​，我们不考虑它的作用区间，而考虑剩下的 1∼31\sim31∼3 个区间。对于这些区间里的数，给它标记上 i+1i + 1i+1。表示这些位置从 i+1i + 1i+1 开始到现在没有被覆盖过。考虑每个位置，它的数字一定是到目前为止最后没有被包含的时刻。

所以对于每组询问 iii，只需要找到第一个标号 ≤li\le l_i≤li​ 的数即可。

当然需要进行一些离散化。可以发现对于每次询问，答案一定在某条线段的端点上，所以关键点设为所有线段端点即可。
